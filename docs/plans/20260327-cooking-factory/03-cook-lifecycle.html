Cooking Lifecycle — Plan Pipeline

How a plan moves through the cooking factory: queue → implement → validate → review → done
Step 1 Plan Queued

📄 Plan added to "Plans Waiting"

URL: docs/plans/auth-module/plan.md

Status: ⏳ Waiting for worker

▼
Step 2 Auto-Assigned

🔥 Idle Worker-1 picks plan from queue

Plan moves to "Being Cooked"

App sends: Implementation prompt + plan URL → PTY

PTY Output — Worker-1
Here is a plan to implement: file://docs/plans/auth-module/plan.md
Read the plan and implement it fully...
> Reading plan...
> Creating src/auth/module.ts
> Writing tests...
▼
Step 3 Idle Polling

⏱️ PTY idle for 60 seconds

App sends: "Where are you up to?" → structured query

Expected response: AIAGENT-FINISHED or AIAGENT-QUESTION

PTY Output — Worker-1
AIAGENT-FINISHED

↳ If AIAGENT-QUESTION → skip to Step 5

▼
Step 4 Validation Phase

🔍 App sends: Validation prompt + same plan URL → PTY

Worker reviews implementation, fixes gaps

PTY Output — Worker-1
Reviewing implementation against plan...
Found 2 missing test cases
Adding tests for edge cases...
All tests pass ✓
AIAGENT-FINISHED
▼
Step 5 Waiting Approval

✋ Plan moves to "Waiting Approval"

Human reviews the finished work:

Review Card
Plan: auth-module/plan.md
Worker: Worker-1 (claude-code)
Branch: feat/auth
Status: ✅ Finished or ❓ Question: "Which auth provider?"
Approve
▼
Send Back
▼
Step 6a Done

✅ Plan disappears from board

Worker goes idle

Auto-picks next queued plan

Step 6b Re-Cook

🔄 Human provides new instructions

Plan returns to "Being Cooked"

Worker resumes with feedback

⚙️
Mandatory review toggle: when ON, all plans must be human-approved before completion. When OFF, auto-finished plans (no questions) skip review entirely — the worker immediately picks the next queued plan.
