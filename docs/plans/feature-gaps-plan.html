Phase 1 — P0: Critical Fixes

1.1 Voice Rip-Out & Hold-to-Passthrough

🔴 Not Started

Problem

src/voice/openwhisper.ts contains 250+ lines of broken audio recording scaffolding (FFmpeg, Sox, PowerShell, whisper.cpp). None of it works in production. The app has no business recording audio — the target CLI app (Claude Code, OpenWhisper, etc.) handles voice natively.

Solution

Delete the entire src/voice/ directory. Replace the voice and openwhisper binding action types with a single hold-key action — a configurable hold-to-passthrough:

  1. Gamepad button pressed → start delay timer
  2. After configurable delay (e.g. 200ms) → send modifier+key combo DOWN on keyboard
  3. Haptic pulse on activation (ties into Phase 2.3)
  4. Gamepad button released → send modifier+key combo UP on keyboard
  5. The OS / target app handles the rest (Claude Code listens for Space hold, OpenWhisper for Ctrl+Alt, etc.)
flowchart LR
    A[B Button Down] --> B[Wait delay\ne.g. 200ms]
    B --> C[Send key combo DOWN\n+ haptic pulse]
    C --> D[Button still held\nkeys stay held]
    D --> E[B Button Up]
    E --> F[Send key combo UP]

The key combo is configurable per binding in the profile YAML:

# config/profiles/default.yaml
cliTypes:
  claude-code:
    B:
      action: hold-key
      keys: [space]           # Claude Code voice = hold Space
      delay: 200
  copilot-cli:
    B:
      action: hold-key
      keys: [ctrl, alt]       # OpenWhisper = hold Ctrl+Alt
      delay: 200

Changes

  • DELETE src/voice/openwhisper.ts and src/voice/index.ts — rip it all out
  • src/output/keyboard.ts — Add keyDown(key) and keyUp(key) methods (robotjs supports this)
  • src/config/loader.ts — Remove openwhisper config type. Add hold-key action with keys and delay fields.
  • config/tools.yaml — Remove openwhisper section entirely
  • config/profiles/default.yaml — Replace voice/openwhisper B-button bindings with hold-key
  • src/index.ts & IPC handlers — Remove all OpenWhisper imports and references

1.2 Session Crash Recovery & Auto-Discovery

🔴 Not Started

Problem

Spawned processes that die leave ghost sessions. On restart, all sessions are lost. No way to rediscover existing terminal windows.

Solution

flowchart TD
    A[App Starts] --> B[Load persisted sessions\nfrom config/sessions.yaml]
    B --> C[For each saved session:\ncheck if window still exists]
    C -->|Window found| D[Restore session]
    C -->|Window gone| E[Discard stale]
    D --> F[Normal operation]
    E --> F
    F --> G[Health check every 5s]
    G -->|alive| F
    G -->|dead| H[Remove + persist]
