Gamepad CLI Hub

Xbox controller to multi-CLI session manager. Control multiple CLI sessions (Claude Code, Copilot CLI, etc.) with a single game controller.

🎮 Gamepad Input
Xbox controller detection with debouncing
🔄 Session Switching
Switch between CLI sessions with D-pad
⚡ Process Spawning
Launch new CLI instances on demand
⌨️ Keyboard Simulation
Send keystrokes to active session
🎙️ Voice Input
Built-in voice or OpenWhisper transcription
⚙️ YAML Config
Customizable button mappings per CLI type

Quick Start

npm install
npm start

Setup Guide

Prerequisites

  • Windows 10 or 11
  • Node.js 18+ installed
  • Xbox controller (wired or wireless)
  • Windows Terminal (recommended)
  • PowerShell Core (pwsh) - included with Windows 11 and Windows 10 1709+

Installation Steps

  1. Clone the repository: 
    git clone <repository-url>
cd gamepad-cli-hub
  2. Install dependencies: 
    npm install
  3. Connect your Xbox controller
  4. Review and customize configuration in config/ (optional - defaults are provided)
  5. Start the application: 
    npm start

Xbox Controller Setup

Your Xbox controller should work automatically once connected via USB or Xbox wireless adapter. The application uses dual input detection methods:

  • Browser Gamepad API - Primary detection method, works with Bluetooth controllers
  • PowerShell XInput - Fallback for wired/Xbox wireless adapter connections
⚠️ Controller Not Detected?

Ensure your controller is properly paired in Windows Settings > Bluetooth & devices > Xbox controller. Press any button to wake the controller from sleep mode.

Windows Terminal Requirements

Windows Terminal is recommended for the best experience. Download it from the Microsoft Store or via winget:

winget install Microsoft.WindowsTerminal

Development Mode

For development with auto-reload on file changes:

npm run dev

Button Mapping Configuration

All button mappings are defined in config/bindings.yaml. Each CLI type can have its own button bindings, and there are also global bindings that work regardless of the active session.

Available Action Types

Action Description Options
keyboard Send keyboard keystrokes to the active window keys: array of key names
voice Trigger built-in voice input (long-press spacebar) None
openwhisper Record audio and transcribe using OpenWhisper recordingDuration: milliseconds
session-switch Switch between active CLI sessions (auto-focuses the window) direction: "previous" or "next"
spawn Launch a new CLI instance cliType: CLI type identifier
list-sessions Display all tracked sessions None
profile-switch Switch to a different binding profile direction: "previous" or "next"
hub-focus Bring the hub window to foreground None

Example Button Mapping

# Face button binding examples
A:
  action: "keyboard"
  keys: ["Clear"]

B:
  action: "openwhisper"
  recordingDuration: 5000

# D-Pad binding (usually in global section)
Up:
  action: "session-switch"
  direction: "previous"

CLI-Specific vs Global Bindings

When a button is pressed, the system first checks for a CLI-specific binding for the active session's CLI type. If none exists, the global binding is used. This means the same button can have different actions depending on which CLI session is active.

Available Buttons

Button Name Description
A, B, X, YFace buttons
LeftTrigger, RightTriggerAnalog triggers
LeftBumper, RightBumperShoulder buttons
LeftStick, RightStickStick click buttons
Up, Down, Left, RightD-Pad directions
Start, BackCenter buttons
GuideXbox logo button (center)

Key Names for Keyboard Action

When using the keyboard action, the keys array accepts:

  • Letters: "a" through "z"
  • Numbers: "0" through "9"
  • Modifiers: "Ctrl", "Alt", "Shift", "Command"
  • Special keys: "Enter", "Escape", "Space", "Tab", "Clear", "Backspace", "Delete"
  • Function keys: "f1" through "f12"
💡 Modifier Keys

For key combinations, include the modifier first: keys: ["Ctrl", "c"] sends Ctrl+C.

Claude Code Integration

Claude Code has excellent built-in voice input via long-press spacebar. Gamepad CLI Hub can trigger this automatically, or you can use OpenWhisper for local transcription.

Recommended Button Setup

Button Action Purpose
A Clear Reset the terminal screen
B Voice (OpenWhisper) Record and transcribe voice input
X /help Show help command
Y /status Show status command

Configuration Example

cliTypes:
  claude-code:
    name: "Claude Code"
    spawn:
      command: "wt"
      args: ["-w", "0", "cc"]
    bindings:
      A:
        action: "keyboard"
        keys: ["Clear"]
      B:
        # Use "voice" for built-in (long-press spacebar)
        # Use "openwhisper" for local transcription
        action: "openwhisper"
        recordingDuration: 5000
      X:
        action: "keyboard"
        keys: ["/", "h", "e", "l", "p", "Enter"]
      Y:
        action: "keyboard"
        keys: ["/", "s", "t", "a", "t", "u", "s", "Enter"]

Voice Input Options

You have two options for voice input with Claude Code:

  • Built-in Voice (action: "voice"): Simulates a long-press of spacebar to trigger Claude Code's built-in voice input system.
  • OpenWhisper (action: "openwhisper"): Records audio and transcribes it locally using whisper.cpp, then types the result.

Copilot CLI Integration

GitHub Copilot CLI (gh copilot) provides AI-powered command suggestions. Gamepad CLI Hub can spawn new instances and send common commands.

Recommended Button Setup

Button Action Purpose
A Clear Reset the terminal screen
B Escape Cancel current operation
X ? + Enter Get help/suggestions
Y Ctrl+C Interrupt command

Configuration Example

cliTypes:
  copilot-cli:
    name: "GitHub Copilot CLI"
    spawn:
      command: "wt"
      args: ["-w", "0", "gh", "copilot"]
    bindings:
      A:
        action: "keyboard"
        keys: ["Clear"]
      B:
        action: "keyboard"
        keys: ["Escape"]
      X:
        action: "keyboard"
        keys: ["?", "Enter"]
      Y:
        action: "keyboard"
        keys: ["Ctrl", "c"]
ℹ️ Difference from Claude Code

Copilot CLI doesn't have built-in voice input like Claude Code. For voice functionality with Copilot CLI, use the openwhisper action.

OpenWhisper Integration

OpenWhisper provides offline speech-to-text transcription using whisper.cpp, a high-performance C++ implementation of OpenAI's Whisper model. It runs entirely locally on your machine—no internet connection required.

What is OpenWhisper?

  • whisper.cpp: C++ implementation of OpenAI's Whisper model
  • Offline: No internet connection needed after setup
  • Privacy: Audio never leaves your machine
  • Fast: Optimized for CPU inference

Installation Steps

  1. Clone whisper.cpp: 
    git clone https://github.com/ggerganov/whisper.cpp.git
cd whisper.cpp
  2. Build the Windows executable: 
    cmake -B build
cmake --build build --config Release

    The executable will be at build/bin/Release/whisper.exe

  3. Download a model file: 
    cd models
.\download-ggml-model.sh base.en

    Or manually download from HuggingFace

  4. Copy files to your project:
    • Copy whisper.exe to your project directory
    • Copy the model file (e.g., ggml-base.en.bin) to models/

Configuration

# OpenWhisper configuration for local transcription
openwhisper:
  whisperPath: "whisper.exe"
  model: "models/ggml-base.en.bin"
  language: "en"
  # tempDir: "/tmp/gamepad-voice"  # Optional: custom temp directory

Configuration Options

Option Description Default
whisperPath Path to whisper.cpp CLI executable "whisper.exe"
model Path to the Whisper model file "models/ggml-base.en.bin"
language Language code or "auto" for auto-detect "en"
tempDir Custom temp directory for recordings System temp
recordingDuration Max recording length in milliseconds (per binding) 5000

Model Choices

Model Description Size Speed
tiny Fastest, least accurate ~39MB ⚡⚡⚡
base Good balance ~74MB ⚡⚡
base.en English-only, better than base for EN ~74MB ⚡⚡
small Better accuracy ~244MB ⚡
medium Very good accuracy ~769MB ~
large Best accuracy, slowest ~1.5GB 🐌

Using OpenWhisper in Button Bindings

bindings:
  B:
    action: "openwhisper"
    recordingDuration: 5000
⚠️ Recording Requirements

OpenWhisper requires a working microphone. Ensure Windows microphone permissions are granted and your default recording device is set correctly.

Language Settings

The language option accepts:

  • "en" - English
  • "es" - Spanish
  • "fr" - French
  • "de" - German
  • "auto" - Auto-detect language
  • And more - see Whisper documentation for full list

Action Details by Type

1. keyboard

Simulates pressing keys on the keyboard.

A:
  action: "keyboard"
  keys: ["Ctrl", "c"]

Options:

  • keys: Array of key names to press in sequence. Modifiers should come before the key they modify.

2. voice

Simulates a long-press of the spacebar (for Claude Code's built-in voice input).

B:
  action: "voice"

Options: None

3. openwhisper

Records audio and transcribes it using OpenWhisper, then types the result.

B:
  action: "openwhisper"
  recordingDuration: 5000

Options:

  • recordingDuration: Maximum recording time in milliseconds (default: 5000)

4. session-switch

Switches the active CLI session and automatically brings the selected window to the foreground.

Up:
  action: "session-switch"
  direction: "previous"

Down:
  action: "session-switch"
  direction: "next"

Options:

  • direction: Either "previous" or "next"

5. spawn

Launches a new CLI instance using the spawn configuration from the specified CLI type.

LeftTrigger:
  action: "spawn"
  cliType: "claude-code"

Options:

  • cliType: The CLI type identifier defined in the cliTypes section

6. list-sessions

Displays all tracked CLI sessions in the console.

Start:
  action: "list-sessions"

Options: None

7. profile-switch

Switches to a different binding profile.

Back:
  action: "profile-switch"
  direction: "previous"

Start:
  action: "profile-switch"
  direction: "next"

Options:

  • direction: Either "previous" or "next"

8. hub-focus

Brings the hub window to the foreground (useful when you've switched focus to a CLI session).

Guide:
  action: "hub-focus"

Options: None

Troubleshooting

Controller Not Detected

Symptoms

Application starts but button presses don't register. Console shows "Connected Gamepads: 0".

Solutions:

  • Verify the controller is connected in Windows Settings > Bluetooth & devices
  • Try a different USB port or wireless pairing method
  • Press any button - the controller may be in sleep mode
  • Test the controller in Windows Game Controller settings (search "Set up USB game controllers")
  • Bluetooth connections can be unreliable; try USB or Xbox Wireless Adapter

Window Focus Issues

Symptoms

Keyboard commands go to the wrong window or don't appear to work.

Solutions:

  • Use D-Pad Up/Down to ensure the correct session is active
  • Press the Start button to list all sessions and verify tracking
  • Close terminal windows that are no longer in use
  • Run as Administrator if window detection is failing

OpenWhisper Recording Problems

Symptoms

Voice input doesn't produce text or fails silently.

Solutions:

  • Verify whisper.exe is in the correct path and runs from command line
  • Check that the model file exists and is accessible
  • Test whisper.exe manually: whisper.exe -f test.wav
  • Ensure Windows microphone permissions are granted
  • Check that no other app is using the microphone
  • Verify your default recording device in Windows sound settings

Spawned Windows Don't Appear

Symptoms

Pressing trigger/bumper doesn't open a new window.

Solutions:

  • Verify the spawn command works manually in a terminal
  • Check that Windows Terminal is installed
  • Review the args in your bindings.yaml configuration
  • For Windows Terminal, ensure -w 0 flag is used for a new window

Debouncing Issues

Symptoms

Single button press triggers multiple actions.

Solutions:

  • This should be handled automatically by the debouncing logic
  • If issues persist, check the controller hardware for sticky buttons
  • Try a different controller to isolate the issue

Configuration Errors

Symptoms

Application fails to start with "Error loading configuration" message.

Solutions:

  • Validate your YAML syntax using an online YAML validator
  • Check for correct indentation (YAML is space-sensitive, use 2 spaces)
  • Ensure all strings are quoted
  • Verify button names match the available buttons exactly
  • Check that action types are spelled correctly
